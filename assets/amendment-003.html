Slide 04 · of 10 · Underlying Magic
Hardware key in a vault.
Touch to sign.
No cloud. No vendor. No API to attack.
  • Each operator holds their own ESP32 + fingerprint sensor + LoRa 915 MHz signing device — ~$40 of hardware, provisioned in their house
  • Private key is burned into eFuse silicon — never extractable by software, requires nation-state-grade attack to physically read
  • Signing requires biometric touch from the operator's own finger — 10 fingers map to 10 different cert actions (full pass / partial pass / revocation / blacklist / kill switch)
  • Device lives in a bolted-down fire safe, antenna passes through coax bulkhead, signing happens over LoRa air-gap to the audit server
  • Five-gate audit pipeline: hash scan → red-team battery → membership inference → dataset attestation → harmful output classifier
  • Cert artifacts are reproducible — any skeptic can re-run the audit and verify the cryptographic signature against the published pubkey
