# Contributing

## Who This Is For

cochranblock is the production site of The Cochran Block, LLC. External contributions are welcome but narrowly scoped — bug fixes, documentation, and improvements that align with the zero-cloud, single-binary philosophy.

## Before You Start

Read [PROOF_OF_ARTIFACTS.md](../PROOF_OF_ARTIFACTS.md) and [TIMELINE_OF_INVENTION.md](../TIMELINE_OF_INVENTION.md) first. They tell you what exists and why decisions were made.

## Code Style

All handlers follow the token-optimized pattern documented in [Code Style](code-style.md). New handlers get the next available `fNN` number. No exceptions.

## Provenance Requirement

Every substantive contribution requires a [TIMELINE_OF_INVENTION.md](../TIMELINE_OF_INVENTION.md) entry with:
- Date
- What was built
- `AI Role` field (none / assisted / generated / reviewed)
- Commit hash

This is a legal requirement, not a preference. See [Provenance](provenance.md).

## Supply Chain

Run `cargo audit` before any PR. Zero unignored advisories required.

```bash
cargo audit
```

## Header Protocol

All `.rs`, `.toml`, `.md`, `.sh` files must have the Unlicense + Contributors header. After editing, run:

```bash
cargo run -p header-writer -- cochranblock
```
